🎯 What This Builds

A Business Analyst sends a requirement in Slack or Google Chat. Ruflo's agent swarm autonomously analyzes the codebase, creates a scope document, waits for your approval, implements the changes, writes tests, pushes a PR, and pings you when done — all without leaving your chat tool.

💡
Why Ruflo over bare Claude Code? Ruflo adds the multi-agent coordination layer — one queen agent orchestrates multiple specialist workers in parallel, with shared memory so the coder knows what the architect planned and the tester knows what the coder changed. It also adds the human-in-the-loop approval gates as a native workflow feature.

⚡ The 7-Step End-to-End Workflow

1
📨 BA Posts Requirement in Slack / Google Chat
The business analyst simply @mentions the bot in a dedicated channel (e.g. #ai-dev-requests) with a natural language requirement. No ticket system, no special format needed.
Slack @ClaudeCode Google Chat @DevBot Plain English input
2
🔍 BA Interpreter + Architect Agents Analyze Codebase
Ruflo spawns two agents in parallel: (a) the BA Interpreter parses the requirement into structured tasks; (b) the Architect reads the GitHub repo via MCP, identifies affected files, and estimates complexity. The HNSW vector memory indexes codebase context at 150x brute-force speed.
GitHub MCP HNSW Vector Search Parallel agents
3
📄 Scope Document Generated + Sent for Approval
The Architect agent produces a structured scope document including: requirement interpretation, affected files, risk assessment, estimated effort, proposed branch name, and a line-by-line change plan. This is posted back to the Slack/Chat thread with interactive Approve / Request Changes / Reject buttons.
ADR format Interactive Slack blocks PDF/MD attachment
🔐
HUMAN APPROVAL GATE #1 — Scope Sign-off
Workflow is fully paused. The agent swarm sleeps until the BA/Tech Lead clicks Approve in Slack or Google Chat. They can also request changes, and the Architect agent will iterate on the scope document.
4
⚙️ Coder Agent Implements Changes
On approval, Ruflo spawns the Coder agent on a fresh feature branch. It works through the approved scope plan file-by-file, using the Filesystem MCP for edits and the GitHub MCP to read existing code. A PostToolUse hook runs the linter after every file edit, catching errors in real-time. Progress updates are streamed back to the Slack thread.
Feature branch Filesystem MCP Live progress in Slack PostToolUse hooks
5
🧪 Tester Agent Runs Tests + Creates Test Case Document
A specialized Tester agent runs the existing test suite, writes new unit/integration tests for changed code, executes them, and records results. It then generates a structured Test Case Document (Markdown + optional DOCX) covering: test scenarios, expected vs actual outcomes, edge cases, and pass/fail summary. CVE scanner also runs on new dependencies.
TDD approach Auto test generation CVE scanning Test Case DOCX
6
📬 DevOps Agent Pushes + Opens Pull Request
The DevOps agent commits all changes, pushes to the feature branch, and opens a GitHub PR. The PR body is auto-populated with: requirement origin, scope doc link, test results, affected files, and a changelog. The PR triggers your existing GitHub Actions CI/CD pipeline automatically.
GitHub MCP Auto PR body CI/CD trigger
7
🔔 User Notified with Full Summary
A final Slack/Google Chat message is posted to the original thread (and optionally a separate #dev-prs channel) with: PR link, test summary, scope doc attachment, and test case document. The BA who raised the ticket gets a @mention. The tech lead gets tagged in the GitHub PR for final review.
PR link + status Documents attached @mention BA + TL

💬 Google Chat Integration

⚠️
Google Chat doesn't have a native Claude Code integration (unlike Slack). You need to build a lightweight bridge. Here's the simplest production-ready approach.

Architecture

  • Google Chat App (Pub/Sub)
  • Cloud Run service (bridge)
  • Triggers Ruflo workflow via webhook
  • Sends updates back via Chat API

Interactive Cards

  • Scope doc as Google Card v2
  • Approve / Reject buttons
  • Progress via card updates
  • Final summary with Drive links
📄 google-chat-bridge/index.ts — Cloud Run Bridge
import { GoogleChat } from '@google-apps/chat';
import { RufloClient } from './ruflo-client';

export async function handleChatEvent(event: ChatEvent) {
  if (event.type === 'MESSAGE') {
    // BA posted a requirement
    const taskId = await RufloClient.startWorkflow({
      requirement: event.message.text,
      requesterId: event.user.email,
      spaceId: event.space.name,
      approvalChannel: 'google-chat'
    });
    return buildProgressCard('Analyzing your repository...', taskId);
  }

  if (event.type === 'CARD_CLICKED') {
    const { action, taskId } = event.action.parameters;
    if (action === 'APPROVE') {
      await RufloClient.approveGate(taskId);
      return buildImplementingCard(taskId);
    }
    if (action === 'REQUEST_CHANGES') {
      await RufloClient.requestScopeRevision(taskId, event.message.text);
    }
  }
}

GitHub Actions — Notify on PR Created

name: Ruflo Workflow Notifications

on:
  pull_request:
    types: [opened, review_requested]
  check_suite:
    types: [completed]

jobs:
  notify:
    runs-on: ubuntu-latest
    if: startsWith(github.head_ref, 'feature/ruflo-')
    steps:
      - name: Notify Slack
        uses: slackapi/slack-github-action@v2
        with:
          channel-id: '#ai-dev-requests'
          payload: |
            {
              "text": "🎉 PR #${{ github.event.pull_request.number }} ready!",
              "attachments": [{
                "color": "good",
                "fields": [
                  {"title": "PR", "value": "${{ github.event.pull_request.html_url }}"},
                  {"title": "Tests", "value": "${{ job.status }}"}
                ]
              }]
            }
        env:
          SLACK_BOT_TOKEN: ${{ secrets.SLACK_BOT_TOKEN }}

CLAUDE.md — Instructions for Repo-Specific Behavior

📄 CLAUDE.md — Place in repo root
# Ruflo Workflow Instructions

## Commit Convention
Use conventional commits: feat:, fix:, test:, docs:, refactor:

## Branch Naming
All Ruflo branches must be: feature/ruflo-{task-description}-{YYYYMMDD}

## PR Requirements
Every PR must include:
- Summary of changes (2-3 sentences)
- Link to original Slack thread
- Test results summary
- Breaking change notice if applicable

## Testing Standards
- Minimum 80% coverage on changed files
- All existing tests must pass
- New tests required for new functions

## Code Style
Run ESLint + Prettier before committing.
Never commit directly to main or develop.

📄 scripts/notify-slack.js — Progress Notification
const { WebClient } = require('@slack/web-api');
const slack = new WebClient(process.env.SLACK_BOT_TOKEN);

const phaseEmojis = {
  'analyzing': '🔍', 'scoping': '📄',
  'implementing': '⚙️', 'testing': '🧪',
  'pushing': '🚀', 'complete': '✅'
};

async function notify() {
  const phase = process.env.RUFLO_PHASE;
  const taskId = process.env.RUFLO_TASK_ID;
  const threadTs = await getThreadTs(taskId); // stored in AgentDB

  await slack.chat.postMessage({
    channel: process.env.SLACK_CHANNEL,
    thread_ts: threadTs,
    text: `${phaseEmojis[phase]} Phase: **${phase}**`
  });
}
notify();

🚀 Quick Setup Checklist

1. Install Ruflo

# Initialize in your repo
npx ruflo@latest init --wizard

# Add MCP servers to Claude Code
claude mcp add ruflo -- npx -y ruflo@latest mcp start
claude mcp add github -- npx -y @modelcontextprotocol/server-github
claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem /workspace

# Verify
npx ruflo doctor --fix
claude mcp list

2. Configure Slack Bot

# Clone the bot (Option B)
git clone https://github.com/mpociot/claude-code-slack-bot
cd claude-code-slack-bot
npm install
cp .env.example .env

# Set in .env:
SLACK_BOT_TOKEN=xoxb-...
SLACK_APP_TOKEN=xapp-...
SLACK_SIGNING_SECRET=...
BASE_DIRECTORY=/workspace/your-repo
DEBUG=true

# Add mcp-servers.json (see above), then:
npm run prod

3. Initialize the Workflow Swarm

# Start the hierarchical swarm
npx ruflo swarm init --topology hierarchical --max-agents 8

# Spawn the hive mind with BA workflow config
npx ruflo hive-mind spawn "BA-to-Deploy Pipeline" \
  --queen-type strategic \
  --config ruflo-workflow.json \
  --consensus byzantine

# Start the daemon (keeps agents running)
npx ruflo daemon start

# Monitor status
npx ruflo status --watch
✅
